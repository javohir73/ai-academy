# Level 6 — Capstone, Portfolio & Career

> Ship a real end-to-end AI product and become hireable. Less new theory, more synthesis, judgment, and communication.

**Goal:** Build and deploy a real AI product, document it honestly, and prepare for the job market.
**Audience:** Learners who finished the relevant track (Levels 0–5 for the full path).
**Prerequisites:** Levels 0–5 (or the track-specific subset).
**Duration:** 4–6 weeks.
**Maps to:** Individual Project + Responsible AI + career prep.

## Learning Outcomes
By the end, a learner can:
- Scope, build, evaluate, and deploy a complete AI project on a real problem.
- Document a model honestly (model card, limitations, failure modes).
- Articulate the *why* behind their technical choices.
- Present a portfolio and pass technical + AI-evaluation interview screens.

## Teaching Approach (this level)
- **Coach, don't lecture** — learners drive; instructor/AI tutor reviews and challenges.
- **"Explain it" is the deliverable** — every artifact must be defended verbally.
- **Honesty over polish** — documenting failure modes scores higher than hiding them.

## Module 6.1 — Capstone Project
**Why this module matters:** the integrative proof that the learner can build *and* explain.

### L6.1.1 — Scoping a real problem & dataset
- **Objective:** choose a tractable, meaningful project.
- **Concepts:** problem framing, data availability, scope control.
- **Hands-on:** write a one-page project proposal.
- **Pitfall:** scope too big to finish.

### L6.1.2 — Build → evaluate → iterate
- **Objective:** execute the full ML loop on a real problem.
- **Concepts:** baselines, iteration, honest evaluation.
- **Hands-on:** ship a baseline, then improve it measurably.
- **Pitfall:** endless tuning with no baseline to beat.

### L6.1.3 — Deploy a demo
- **Objective:** make the model usable by others.
- **Concepts:** Gradio/Streamlit, simple deployment.
- **Hands-on:** wrap the model in an interactive demo.
- **Pitfall:** a model nobody but the author can run.

### L6.1.4 — Write it up
- **Objective:** communicate the work.
- **Concepts:** model card, results, honest limitations.
- **Hands-on:** produce a README + model card.
- **Pitfall:** overclaiming performance.

## Module 6.2 — Responsible AI in Practice
**Why this module matters:** turns the responsible-AI thread into shippable artifacts.

### L6.2.1 — Data ethics & privacy
- **Objective:** handle data responsibly.
- **Concepts:** consent, PII, privacy basics.
- **Hands-on:** audit your dataset for privacy issues.
- **Pitfall:** using data you have no right to.

### L6.2.2 — Documenting bias & failure modes
- **Objective:** be transparent about limits.
- **Concepts:** subgroup performance, known failure cases.
- **Hands-on:** add a "limitations" section with evidence.
- **Pitfall:** reporting only aggregate metrics.

### L6.2.3 — Model cards & datasheets
- **Objective:** standardize documentation.
- **Concepts:** model card and datasheet templates.
- **Hands-on:** complete a model card for your capstone.
- **Pitfall:** filling it in as box-ticking, not insight.

## Module 6.3 — Career & Interview Prep
**Why this module matters:** competence isn't enough if you can't show it.

### L6.3.1 — Building an AI portfolio
- **Objective:** present your best work.
- **Concepts:** project selection, GitHub hygiene, writeups.
- **Hands-on:** assemble a portfolio site/repo.
- **Pitfall:** many half-finished projects vs a few polished ones.

### L6.3.2 — Explaining your projects
- **Objective:** articulate decisions clearly.
- **Concepts:** the "explain it" skill, trade-off narration.
- **Hands-on:** record a 3-minute project walkthrough.
- **Pitfall:** describing what, not why.

### L6.3.3 — Mock technical interview
- **Objective:** practice under realistic conditions.
- **Concepts:** ML fundamentals Q&A, AI-evaluation / rubric-style screens (e.g., Micro1/Zara format).
- **Hands-on:** timed mock interview + feedback.
- **Pitfall:** memorizing answers vs reasoning aloud.

### L6.3.4 — Staying current
- **Objective:** keep learning in a fast field.
- **Concepts:** reading papers, following releases, evaluating hype.
- **Hands-on:** set up a personal "stay-current" routine.
- **Pitfall:** chasing every new model and finishing nothing.

---

## Final Deliverable
- **Portfolio:** 3–5 polished projects (roughly one per level completed) + one flagship capstone, each with code, a writeup, and a live demo. Each must be defensible in a 3-minute verbal walkthrough.

## Mastery Check (graduation bar)
| Skill | Pass bar |
|-------|----------|
| Build | A working, deployed capstone on a real problem |
| Document | Complete model card with honest limitations |
| Explain | Pass a mock interview reasoning aloud, not reciting |
| Responsible AI | Identify and document your model's failure modes |

## Tools & Environment
- Full stack: PyTorch / Hugging Face / scikit-learn, Gradio or Streamlit, GitHub.

## Outcome
→ Job-ready in applied AI/ML, with a portfolio that demonstrates *build AND explain*.
